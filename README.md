# portfolio-backend

## Why this chatbot was created

This chatbot was built by Ardhendu to showcase his skills in modern AI and backend development. It serves as an interactive, recruiter-friendly portfolio assistant, allowing users to ask questions about Ardhendu's experience, skills, and projects. The chatbot demonstrates practical implementation of Large Language Models (LLMs) using FastAPI and the latest vectorless RAG architecture.

## How it works & Vectorless RAG

Instead of using traditional vector databases or embeddings for document retrieval, this chatbot uses a vectorless RAG (Retrieval-Augmented Generation) approach. All resume information is extracted from a PDF and provided directly to the LLM in the prompt. The LLM itself performs retrieval and generation in one step, answering user questions based only on the resume content. This simplifies the architecture and leverages the LLM's reasoning abilities for small/medium datasets.

## How chat streaming works

The backend uses FastAPI's `StreamingResponse` to stream answers to the frontend as they are generated by the LLM. When a user asks a question, the backend sends the prompt to Groq's LLM API with streaming enabled. As the LLM generates the answer, each chunk is sent to the frontend in real time, allowing users to see the response as it is being composed. This provides a fast and interactive chat experience.

Summary

The backend sends a streaming request to Groq.
Reads and parses each chunk as it arrives.
Yields each chunk to FastAPI, which streams it to the client.
The user sees the answer appear in real time.
